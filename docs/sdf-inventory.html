7. Process Flow Diagram

Three end-user actions, traced step by step through the records and scripts that carry them. Use this when supporting users who hit an error, or when explaining what's happening behind the schedule editor or the reports.

Flow A User edits a timing schedule on a transaction

  1. User opens a Purchase Order, Sales Order, or BC Change Request.

  2. User Event script fires on beforeLoad.

    • UEcustomscript_bc_cost_timing_ue
    • UEcustomscript_bc_rev_timing_ue
    • UEcustomscript_bc_co_timing_ue

  3. UE stamps the schedule editor HTML into the INLINEHTML mount field on the Cash Flow subtab.

    • Inline HTMLcustbody_bc_cost_timing_html
    • Inline HTMLcustbody_bc_rev_timing_html
    • Inline HTMLcustrecord_bc_co_timing_html

  4. User picks a distribution, clicks Generate, edits rows, or clicks Rebalance / Save. The editor JS runs locally; the math comes from the helper modules.

    • Modulebc_cf_calculator.js
    • Modulebc_timing_engine.js
    • Modulebc_timing_ui.js

  5. Editor JS posts to the Timing Data Suitelet for save and load.

    • Suiteletcustomscript_bc_timing_data_sl

  6. Timing Data Suitelet uses the DAO module to read and write the timing line records.

    • Modulebc_timing_dao.js
    • Recordcustomrecord_bc_cost_timing_line
    • Recordcustomrecord_bc_revenue_timing_line
Flow B User opens Cash Flow reports on a BC Project record

  1. User opens a BC Project record.

    • Custom Segmentcustomrecord_cseg_bc_project

  2. Project User Event fires on beforeLoad.

    • UEcustomscript_bc_cf_project_ue

  3. UE stamps 3 iframes into the 3 INLINEHTML fields on the Cash Flow subtab.

    • Inline HTMLcustrecord_bc_cf_combined_html
    • Inline HTMLcustrecord_bc_cf_cost_html
    • Inline HTMLcustrecord_bc_cf_revenue_html

  4. Each iframe loads its report Suitelet, which returns a skeleton HTML page.

    • Suiteletcustomscript_bc_cf_combined_sl
    • Suiteletcustomscript_bc_cf_cost_report_sl
    • Suiteletcustomscript_bc_cf_rev_report_sl

  5. Client JS in each iframe fetches JSON from the Data Endpoint Suitelet.

    • Suiteletcustomscript_bc_cf_data_sl

  6. Data Endpoint runs SuiteQL over the timing line records and returns aggregated JSON.

    • Recordcustomrecord_bc_cost_timing_line
    • Recordcustomrecord_bc_revenue_timing_line

  7. Client JS renders KPIs, chart, and source-grouped table using the shared UI module.

    • Modulebc_cf_ui.js
    • Modulebc_cf_styles.js
Flow C User opens the cross-project Portfolio report

  1. User clicks BlueCollar → Project Control Center → Cash Flow Portfolio.

  2. Portfolio Suitelet loads and returns a skeleton page.

    • Suiteletcustomscript_bc_cf_portfolio_sl

  3. Client JS fetches JSON from the Data Endpoint using action=portfolio and any active filters.

    • Suiteletcustomscript_bc_cf_data_sl

  4. Data Endpoint rolls up every BC project's timing lines into one response.

    • Custom Segmentcustomrecord_cseg_bc_project
    • Recordcustomrecord_bc_cost_timing_line
    • Recordcustomrecord_bc_revenue_timing_line

  5. Portfolio client JS renders the filter pill, KPIs, paired-bar chart, and one-row-per-project table.
